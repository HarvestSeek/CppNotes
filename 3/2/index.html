    2.std::vector

    1. 定义与初始化

    C++ 中既有类模板，又有函数模板，其中 vector 是一个类模板。

    ways of initializing a vector
    vector<T> v1; 默认初始化，空 vector
    vector<T> v2(v1); v2 包含 v1 中所有的副本，两者类型必须相同
    vector<T> v2 = v1; 等价于 v2(v1)
    vector<T> v3(n, val); 包含 n 个元素，每个的值为 val
    vector<T> v4(n); 包含 n 个执行了值初始化的对象，int 自动设为 0，如果是类则由类默认初始化
    vector<T> v5{a, b, c, ...} 按照花括号中的值初始化
    vector<T> v5 = {a, b, c, ...} 与上面的情况等价

    2. 操作

    operations of vector
    v.empty() 是否为空
    v.size() 元素个数
    v.push_back(t) 尾端添加元素
    v[n] 取 v 中第 n 个位置上的元素的引用
    v1 = v2 用 v2 中的元素拷贝替换 v1
    v1 = {a, b, c, ...} 用列表中的元素拷贝替换
    v1 == v2 true: 当且仅当元素数量相同且对应位置的元素值相同 (元素可比较)
    >, >=, <, <= 字典序比较

    需要注意以下几点：

    • 不能再 for-each 循环中添加或删除元素，因为它预存了容器的大小；
    • 访问 size_type 需正确指明类型，如 vector<int>::size_type；
    • 不能通过下标添加未开辟空间的元素。

    3. Vector 内存分配

     为了支持快速随机访问，vector 对象存储在一块连续的地址空间中。随着容器大小的不断改变，其地址空间也要重新分配，以下接口体现了这个特性。

    manage the size of container
    c.capacity() 不重新分配内存的话，可以保存多少元素
    c.reserve(n) 分配至少能容纳 n 个元素的内存空间
    c.shrink_to_left() 将 capacity() 减少为与 size() 等同的大小

    只有在插入或 resize 时，容器大小达到 capacity，内存才会被重新分配，分配多少取决于具体实现。实际中可根据具体的情况减少频繁的内存分配。
