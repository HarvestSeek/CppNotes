    6.associative container

    1. 关联容器概述

    C++ 标准库提供 8 个关联容器，它们根据是否要求元素重复、关键字是否有序分为三个维度。

    type of associative containers
    map multimap unordered_map unordered_multimap
    set multiset unordered_set unordered_multiset

    map 和 multimap 定义在 <map> 中，set 和 multiset 定义在 <set> 中，无序容器定义在 <unordered_map> 和 <unordered_set> 中。

    2. 定义和初始化

    同其他容器，支持默认初始化、同类型容器拷贝、值范围初始化、值初始化。

    (1). 有序容器的关键字类型

     C++ 算法和容器内通过运算符 < 比较，当且仅当 !(a < b) && !(b < a) 时认为二者相等。因此，key 需要重载 <，并在此运算符上定义一个严格弱序关系 (strict weak ordering)，即满足：① 反自反；② 反对称；③ 传递。如果不满足，则比较或相等的判定 (例如使用 <= 时) 就会产生问题。

    3. 关联容器操作

    (1). pair 类型

    pair
    pair<T1, T2> p; 对两个成员分别进行值初始化
    pair<T1, T2> p(v1, v2);
    pair<T1, T2> p = {v1, v2};    		 根据 v1 和 v2 初始化
    make_pair(v1, v2) 返回一个用 v1 和 v2 初始化的 pair
    p.first, p.second public 成员
    ==, != first 和 second 分别相等
    >, >=, <, <= 字典序比较，均通过 < 实现

    (2). 关联容器定义类型

    type of elements
    key_type 容器的关键字类型。
    mapped_type 每个关键字关联的类型，只适用于 map。
    value_type 对于 set，与 key_type 相同；对于 map，为 pair<const key_type, mapped_type>

    (3). 插入元素

    insert
    c.insert(v)
    c.emplace(args)    		 v 是 value_type，返回一个 pair，包括指定元素的迭代器和插入是否成功的布尔值。
    c.insert(b, e)
    c.insert(i1)    		 插入指定范围或列表的元素，返回 void
    c.insert(p, v)
    c.emplace(p, args)    		 将迭代器 p 作为一个提示，指出从哪开始搜索新元素应该存储的位置。

    (4). 删除元素

    delete
    c.erase(k) 删除每个值为 k 的值，返回数量，类型为 size_type。
    c.erase(p) 删除迭代器 p 指定的元素，返回 p 之后的元素的迭代器。
    c.erase(b, e) 删除范围内元素，返回 e

    (5). map 的下标操作

    map 和 unordered_map (非 const) 提供了下标操作和 at() 函数。multi 容器由于一个 key 可能有多个 value，因此不支持。

    index of map
    c[k] 返回关键字为 k 的元素；如果 k 不在 c 中，添加一个值为 k 的元素，并进行值初始化。
    c.at(k) 若 k 不在 c 中，抛出 out_of_range 异常。

    (6). 访问元素

    access
    c.find(k) 返回指向容器中第一个关键字为 k 的元素的迭代器，不存在则返回尾后迭代器。
    c.count(k) 返回关键字等于 k 的元素的数量。对于不允许重复的容器可能与 find() 没区别。
    c.lower_bound(k) 返回一个迭代器，指向第一个不小于 k 的元素，否则返回尾后迭代器。
    c.upper_bound(k) 返回一个迭代器，指向第一个大于 k 的元素，否则返回尾后迭代器。
    c.equal_range(k) 返回包括两个迭代器的 pair，表示 k 的范围，若不存在两个成员均等于 c.end()

    lower_bound() 和 upper_bound() 不支持无序容器。

    (7). 迭代器

    在解引用关联容器的迭代器时，会得到一个类型为容器的 value_type 的引用。

    • 对 map，返回的是一个 pair，first 是一个 const 的 key；second 为 value 且可以修改；
    • 对 set，虽然它同时定义了 const_iterator 和 iterator，但两种类型都是只读的。
    由于 const 这一特性，关联容器只可用于只读的泛型算法。

    4. 无序容器

     无序容器基于哈希表而不是比较运算符组织元素。无序容器在存储上组织为一组桶，每个桶保存零个或多个元素，它们的哈希值都相同；为了访问一个元素，容器会首先计算哈希值以定位桶，如果允许重复关键字，所有元素都在一个桶中。

    manage the buckets
    bucket interface
    c.bucket_count() 正在使用的桶的数目
    c.max_bucket_count() 最多能容纳的桶的数量
    c.bucket_size() 第 n 个桶有多少元素
    c.bucket(k) 关键字为 k 的元素在哪个桶中
    bucket iteration
    c.begin(n), c.end(n) 桶 n 的首元素迭代器和尾后迭代器
    c.cbegin(n), c.cend(n) const
    hashing strategy
    c.load_factor() 实际每个桶的平均元素数量，返回 float
    c.max_loat_factor() c 试图维护的每个桶的平均元素数量，返回 float
    c.rehash(n) 重组存储，使得 bucket_count>=n && bucket_count>size / max_load_factor
    c.reserve(n) 重组存储，使得 c 可以保存 n 个元素且不必 rehash

    无序容器需要存储类型重载哈希函数和 ==。
