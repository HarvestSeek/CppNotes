    2.variable

    1. 变量初始化

    在 C++ 中，初始化是一个复杂的问题，初始化和赋值是两个不同的操作，但在很多其他的语言中二者的区别可以忽略不计。

    • 初始化：创建变量时赋予一个初始值；
    • 赋值：把对象的当前值擦除，并以新值替代。

    (1). 列表初始化 （list initialization)

    可通过以下四种方式初始化一个变量： 

    1
2
3
4
    int temp = 0;
int temp = {0};
int temp{0};
int temp(0);
    通过花括号初始化即列表初始化，是 C++ 11 的新特性。其重要特点：当初始值存在丢失的风险，编译器会警告。 
    1
2
3
    long double id = 3.1415926536;
int a{id}, b = {id}; // warning
int c(id), d = id;   // correct

    (2). 默认初始化

    定义在函数体外的变量默认值为 0，函数体内部的内置类型如果违背初始化，其值如果被访问将会出错。

    2. 复合类型

     复合类型 (compound type) 是指基于其他类型定义的类型，实现对其他类型的间接访问。C++ 中有多种复合类型。在这里只介绍指针和左值引用。
     <<C++ Primer>> 对 对象 的定义：对象是指一块能存储数据并具有某种类型的内存空间。

    (1). 指针

    即通过 & 取引用，通过 * 解引用以访问原对象的值。其中 void* 是一种特殊的指针类型，可以存放任意对象的地址。从 void* 的视角来看，内存空间仅仅是内存空间，无法访问其中的对象。
    在 C++ 中，有以下三种生成空指针的方式： 

    1
2
3
    int *p1 = nullptr;
int *p2 = 0;
int *p3 = NULL;
     在 C 语言中，NULL 被定义为 #define NULL ((void*)0)，但在 C++ 的 cstdlib 中，它被定义为 #define NULL 0。原因在于，C++ 是强类型的，不允许 C 语言中 void* 指针的隐式转换。
     nullptr 是 C++ 11 引入的新特性，应尽可能使用它初始化空指针而非 0，nullptr 可以自动转换为任意的指针类型，而 0 既是数字又可以是指针，在函数重载时可能会造成歧义。

    (2). 引用

    引用有以下几个特性：

    • 引用必须被初始化，即与一个对象绑定，相当于对象的一个别名，绑定关系不可修改；引用只能与对象绑定，不能是字面值或表达式；
    • 对引用的赋值和访问，相当于对原对象的赋值和访问；
    • 引用本身不是对象，不能生成引用的引用。
    引用可以理解为通过指针常量实现，是一种特殊的指针。

    指针的引用： 

    1
2
3
4
5
    int i = 1024;
int *p;
int *&r = p;
r = &i;
// the type of r is determined by the nearest operator to r

    3. const 限定符

    即限定一个变量初始化后不可被改变。如： 

    const int BufSize = 512;
    在编译时，编译器会把所有用到 BufSize 的地方都替换为相应的值。如果需要在其他文件中使用，则需要在 const 前加 extern。

    (1). const 引用

     const 引用常简称为“常量引用”，指把引用绑定在 const 对象上。 

    1
2
    const int c = 1024;
const int &c1 = c;
     const 引用也可能绑定一个非 const 的对象，原对象的值不可通过此引用修改。但非 const 的引用不能绑定一个 const 对象。

     由于一般的引用可以对原对象进行修改，因此引用的类型要与原对象一致且不能绑定字面量和表达式，const 引用无需此限制。 

    1
2
3
4
5
6
7
    // case1
int i = 42;
const int &i1 = 42;
const int &i2 = r1 * 2;
// case2
double val = 3.14;
const int &v1 = val;
    在情况 2 中，编译器为了让 v1 绑定一个整数对象，创建了一个临时量： 
    1
2
    const int temp = val;
const int &v1 = temp;

    (2). const 与指针

     * 在 const 之后代表这是一个指向常量的指针，从右向左解读，const 限定的部分为“解引用”，即无法通过指针修改原对象的值。需要注意的是，const 对象只能被 const 的指针指向。 

    1
2
3
4
5
6
    const int i = 1024;
const int *i1 = &i;   // <==> int const *i1 = &i;
int *i2 = &i; // ERROR

int j = 1024;
const int *j1 = &j;
     * 在 const 之前代表这是一个常量指针，从右向左解读，此时 const 限定的部分为这个变量，即指针的值不可变，但指针可以指向常量或非常量。 
    1
2
3
4
    int i = 1024;
int *const i1 = &i;

const int *const comb = &i;

    (3). 顶层 const

     顶层 const (top-level const) 表示指针本身是常量或某个对象是常量，而底层 const (low-level const) 表示指针所指的对象是常量。所有的 const 引用都是底层 const。 

    1
2
3
4
5
6
    int i = 0;
int *const p1 = &i; // top-level
const int c = 42;   // top-level
const int *p2 = &c; // low-level
const int *const p3 = p2; // top + low
const int &r = i;   // low-level
    执行对象的拷贝操作时，顶层 const 没有什么问题，底层 const 需要考虑转换。一般的判断方法是，(对于底层 const) 常量不能转换为非常量。 
    1
2
3
4
5
6
    int *px = p1; 
int *py = p3; // ERROR
p2 = p3; 
p2 = &i; // non-const to const
int &r1 = c; // ERROR
const int &r2 = i;
    p3 既是顶层 const 又是底层 const，拷贝时只需考虑底层 const 的性质。

    (4). 常量表达式

     常量表达式 (const expression) 是指之不会改变并且在编译过程中就能得到结果的表达式。字面量、用常量表达式初始化的 const 对象都是常量表达式。 

    1
2
3
    const int max_files = 20;        // const exp
const int limit = max_files + 1; // const exp
const int size = get_size();     // non-const exp
     在一个复杂的系统中，常量表达式时难以分辨的。C++11 规定，允许将变量声明为 constexpr，为变量增加 const 属性，并让编译器验证是否为常量表达式。
     constexpr 限制可声明的类型为字面值类型 (literal type)，包括算术类型、指针和引用，不包括自定义类和 string 类等。
     对于通过函数初始化的 const 变量，如果函数是一个 constexpr 函数 (函数的形参和返回值都是字面值类型，且函数的 return 语句只有一个)，并且函数的实参都是常量表达式，则相应的变量也是常量表达式。

    1
2
3
4
5
6
7
8
9
    constexpr int scale(int arg) {
    return arg * 2;
}

int main() {
    constexpr int val1 = scale(1);
    int i = 100;
    constexpr int val2 = scale(i); // ERROR
}

    对于指针来说，需要注意：

    • 函数体内定义的变量一般来说放在非固定地址中，constexpr 指针不能指向这样的变量；
    • 在 constexpr 声明后定义了一个指针，则只是生成了一个顶层 const，即常量指针。
    1
2
3
4
5
6
    constexpr int i = 42;
int j = 41;
// i and j must be defined out of function
constexpr const int *p1 = &i;
constexpr int *p1 = &j; 
// ERROR: constexpr int *p1 = &i
