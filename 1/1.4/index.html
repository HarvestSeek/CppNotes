    4.array

    1. 定义和初始化

    在 C++ 中，数组的维度在编译时必须已知，维度必须是一个常量表达式。 

    1
2
3
4
    unsigned int = 42;
constexpr unsigned sz = 42;
int *p[sz];
string bad[cnt]; // 可能会编译不通过或产生其他问题
    列表初始化：

    1
2
    const unsigned sz = 6;
int ial[sz] = {0, 1, 2};
    字符数组的特殊性： 
    1
2
3
4
    char a1[] = {'C', '+', '+'};       // 无空字符
char a2[] = {'C', '+', '+', '\0'}; // 显式空字符
char a3[] = "C++";                 // 自动添加空字符
const char a4[6] = "Daniel";       // ERROR
    更加复杂的数组声明： 
    1
2
3
4
5
    int arr[10] = {0, 1, 2,}; 
int *ptrs[10];             // ptrs 是含有10个整形指针的数组
int (*parray)[10] = &arr;  // parray 指向一个含有10个整数的数组
int (&arrref)[10] = arr;   // arrref 引用一个含有10个整数的数组
int *(&arry)[10] = ptrs;   // arry 是数组的引用，该数组含有10个指针
     默认情况下，类型修饰符从右向左依次绑定。对于 ptrs 来说，可以依次知道它是一个大小为 10 的数组、名字为 ptrs、存放类型为 int 的指针。
     对于 parray，由于括号的存在，需要从内向外解读。首先由 *parray 知道它是一个指针，然后依次知道它指向一个大小为 10 的数组、数组中类型为 int。对于 arrref 也是同理。

     在使用数组下标时，常使用 size_t 类型。它是一种机器相关的无符号类型，它被设计的足够大以容纳内存中任意对象的大小。它定义在 cstddef 中，是 C 标准库 stddef.h 的 C++ 版本。

    2. 指针与数组

    在用到数组名字的地方，编译器会自动将其替换为指向数组首元素的指针： 

    string *p2 = nums;  // <==> p2 = &nums[0]
    当使用 decltype 关键字时，上述转换不会发生。 
    1
2
3
4
5
    int ia[] = {0, 1, 2, 3};
// ia2 是一个整型指针，指向 ia 第一个元素
auto ia2(ia); // <==> auto ia2(&ia[0])
// ia3 是一个含有 10 个整数的数组
decltype(ia) ia3 = {0, 1, 2, 3};

    3. 指针与迭代器

     对迭代器的运算也适用于指针。尽管可以通过计算得到尾后指针，但易出错，C++11 在标准库中引入了 begin 和 end 函数。 

    int *pbeg = begin(arr), *pend = end(arr);
    指针相减的结果是一种名为 ptrdiff_t 的类型，和 size_t 定义在同一位置。需要注意的是，不同类型的指针的比较和相减没有意义。

    4. 指针与下标

    只要指针指向的是数组中的元素，就可以执行下标运算。 

    1
2
3
4
    int ia[] = {0, 2, 4, 6, 8};
int *p = &ia[2];
int j = p[1];    // = *(p + 1) = ia[3]
int k = p[-2];   // ia[0]
    标准库的下标运算必须是无符号类型，而内置的下标运算无此要求。
