    3.type

    1. 类型别名

     类型别名是某种类型的同义词。传统的方法是使用 typedef 关键字，新标准使用别名声明定义类型的别名： 

    1
2
3
4
5
    typedef double wages;
typedef wages *p;

using hour = int;
using day = int*;

    如果某个别名类型指代的是复合类型或常量，就可能产生预料不到的后果

    typedef char *pstring;
const pstring cstr = 0;
    const char *cstr:  它的数据类型是 char，"*" 是声明符的一部分。
    上面的写法 :  * 是类型中的一部分，cstr 是一个常量指针。

    2. auto 类型说明符

    有时无法知道用以声明的变量类型。C++11 引入了 auto 说明符，让编译器通过变量的初值分析表达式所属的类型。 

    auto item = val1 + val2; // item must be initialized
    ① 如果以引用初始化，编译器以被引用对象的类型作为 auto 的类型； 
    int i = 0, &r = i;
auto a = r;
    ② auto 一般会忽略顶层 const，保留底层 const，如果希望推断的类型是顶层 const，则需要明确指出。 
    int i = 0;
const int ci = i, &cr = ci;

auto b = ci; // top-level const is ignored
const auto bi = ci;

auto d = &i;
auto e = &ci; // low-level const is reserved

    3. decltype 类型指示符

    有时希望通过一个表达式推断出要定义的变量的类型，但是不希望用这个表达式初始化变量。为此，C++11 引入了 decltype。

    decltype 对于类型的处理和 auto 有些不同： 

    // 1.
const int ci = 0, &cj = ci;
decltype(ci) x = 0; // x: const int
decltype(cj) y = x; // y: const int&, must be inited
// 2.
int i = 42, *p = &i, &r = i;
decltype(r + 0) b;
// 3.
decltype(*p) c; // ERROR: c is int&
// 4.
decltype(i) d;   // d is int
decltype((i)) e; // ERROR: e is int&

    • 1. 如果 decltype 使用的表达式是单变量，会保留顶层 const 和引用的性质；
    • 2. r + 0 的结果是一个 int，因此 b 是一个 int；
    • 3. 如果 decltype 的表达式是一个解引用操作，则获得的是引用类型；
    • 4. 加上括号的变量被当做表达式处理。
