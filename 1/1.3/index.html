    3.type

    1. 类型别名

     类型别名是某种类型的同义词。传统的方法是使用 typedef 关键字，新标准使用别名声明定义类型的别名： 

    1
2
3
4
5
    typedef double wages;
typedef wages *p;

using hour = int;
using day = int*;
    如果某个别名类型指代的是复合类型或常量，就可能产生预料不到的后果。例如： 
    1
2
    typedef char *pstring;
const pstring cstr = 0;
    上面的写法不等同于 const char *cstr，它的数据类型是 char，* 是声明符的一部分。而上面的写法 * 是类型中的一部分，cstr 是一个常量指针。

    2. auto 类型说明符

     在把表达式赋给变量的时候，常常需要知道变量的类型用以声明，但有时做不到。C++11 引入了 auto 说明符，让编译器通过变量的初值分析表达式所属的类型。 

    1
    auto item = val1 + val2; // item must be initialized
    ① 如果以引用初始化，编译器以被引用对象的类型作为 auto 的类型； 
    1
2
    int i = 0, &r = i;
auto a = r;
    ② auto 一般会忽略顶层 const，保留底层 const，如果希望推断的类型是顶层 const，则需要明确指出。 
    1
2
3
4
5
6
7
8
    int i = 0;
const int ci = i, &cr = ci;

auto b = ci; // top-level const is ignored
const auto bi = ci;

auto d = &i;
auto e = &ci; // low-level const is reserved
