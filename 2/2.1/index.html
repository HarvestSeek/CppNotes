    2. 引用形参和指针形参

     形参是函数内的临时变量，和一般的变量初始化一样，拷贝不会影响原值。若需要修改原对象，可以设置形参为引用或指针，二者区别不大。
     当不需要修改原对象，通过指针或引用传入可以避免对原对象进行拷贝 (部分对象甚至不支持拷贝)，节省一部分空间。此时最好使用常量引用，好处在于：

    • 可以直接传入对象，不必取引用
    • 限制对原对象修改
    • 普通引用会限制 const 对象、字面量、需要类型转换的对象的传入

    3. 数组形参

    由于数组不能拷贝，且数组标识符的含义即是指针，因此传参只能通过指针。下面三种形式是等价的，由于参数的类型表达式是指针，第二种形式的维度可以省略。 

    void print(const int*);
void print(const int[]);
void print(const int[10]);
    当多维数组作为形参时，实际上传递的是最外层一维数组首元素的指针，此时第二维及其之后的维数都不能省略，它们是外层数组元素类型表达式的一部分。

    4. 可变参数

    C++11 提供了以下方法实现可变参数

    • 若所有实参类型都相同，可使用 initializer_list，它定义在同名的头文件中
    • 若实参的类型不同，可以编写一个可变参数模板
    • 与 C 接口交互的程序中，一般使用省略符 ...

    initiallizer_list
    initializer_list<T> lst; ...
    lst.begin() lst.end()
    lst.size

    对于 initializer_list：

    • 传递实参时要写在花括号内
    • initializer_list 中的元素是常量，且只能通过迭代器访问

    5. 默认实参

    • 默认实参可以用表达式去初始化，但表达式的值是运行时确定的
    • 默认实参只需声明一次，且放在参数表的最后
      • 严格来说，同一函数如果默认实参不同可以声明多次，但需要保证每个声明的默认实参正交、被声明的默认实参加起来连续地位于参数表的末尾

    (3). 返回一个数组

    和数组作为参数的情况类似，由于数组不能拷贝，数组也不能作为返回值，但是可以返回一个指向数组的指针。其声明方式如下:

    ① 类型别名

    typedef int arrT[10];
// or
using arrT = int[10];
arrT* func(int i);

    ② 直接声明 

    int (* func(int i))[10];

    ③ 尾置返回类型 

    auto func(int i) -> int (*)[10];
