    2.overload and others

    1. 重载

    • 别名和原名之间无法区分函数
    • 顶层 const 和原类型之间无法区分函数，底层 const 可以
    • 返回值无法区分重载函数

    (1). const_cast

    const_cast 可以解除指针或引用的顶层 const，或者为其加上顶层 const 属性。
    对于去除顶层 const，这种去除只是为了函数传参而进行的"表面去除"，若用去除后的变量修改原对象，其行为是未定义的。

string &shorterString(string &s1, string &s2) {
    auto &r = shorterString(const_cast<const string&>(s1), 
                            const_cast<const string&>(s2));
    return const_cast<string&> r;
}
    上面的例子中，shorterString 希望当参数是非常量时，返回值也是非常量，这里正适合使用 const_cast。

    2. 内联函数

     对于一个体积较小且被频繁调用的函数，可以通过内敛优化的方式将其嵌入到被调用处，减小不必要的栈开销。对于体积较大的函数，强制内联会使得目标代码膨胀，从而使得目标文件过大。
     内联的权衡由编译器决定，C++ 可以通过 inline 关键字向编译器提出内联的"建议"，对于现代编译器来说，inline 关键字对于内联优化的意义已经不大，不管其声明与否，可能都对编译器的内联优化行为没什么影响。

    ①. 对于每个需要使用 inline 函数的 translation unit，在当前文件内都要看到此 inline 函数的 definition。并且，每个 unit 内的内联函数定义都不会生成外部符号 (类似于 static)。

    // test.h
void test();

// test.cpp
inline void test() { }

// main.cpp
#include "test.h"
int main() {
    test();
}
    在这个例子中，链接器会因为找不到 test() 的定义而报错。

    ②. 出于对 ① 的考虑，C++ 允许 inline 函数的多个同名定义存在，即编译器以某种机制告诉链接器不报内联函数同名的错误，而调用的时候实际执行哪个定义是不被确定的。

    因此，通常将 inline 函数的定义直接写在头文件中。并且，如果在类的作用域内直接写函数的定义，会默认为其加上 inline 关键字。
