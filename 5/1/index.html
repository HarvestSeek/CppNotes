    1.constructor

    1. 构造函数

    (1). 默认构造函数

     类通过默认构造函数执行默认初始化的过程，当一个类没有任何构造函数时，编译器会创建一个默认构造函数，它被称为合成的 (synthesized) 默认构造函数：
     ① 如果存在类内值，则用类内值初始化； ②否则，执行相应成员的默认初始化。
    当类中存在自定义的构造函数，且合成的默认构造函数也被需要时，需要显式声明： 

    class A {   A() = default;  }
    合成的默认构造函数只适用于非常简单的类，应尽可能自定义默认构造函数。

    (2). 构造函数初始值列表

    即通过如下形式初始化成员: 

    void func(<params>) : member1(expr), ... { }
    类中的所有成员根据它们在类的定义中出现的顺序进行初始化，每个成员依次按照 构造列表 -> 类内值 -> 默认构造函数的顺序寻找初始化的机会。因此，构造函数初始值列表的必要性在于：
    • 是通过参数初始化 const 类型和引用类型的唯一机会；
    • 一些类型会涉及初始化与赋值的底层效率问题。

    当一个成员的初始化需要用到另一个成员时，需要注意它们在类中定义的顺序。

    (3). 委托构造函数

    把自己的一部分职责委托给其他构造函数。 

    struct A {
    char a, b;
    A(char i, char j) : a(i), b(j) { }
    A(string s) : A(s[0], s[1]) { }
};

    (4). 隐式类类型转换

     只需要一个实参的构造函数被称为转换构造函数，它隐式地定义了由其他类型转换到此类型的规则，编译器会根据相应参数和构造函数自动创建一个临时量。 

    struct A {
    string label = "succ";
    A(string s) { }
    void func(A fake) { cout << fake.label; }
};

int main() {
    A a("a");
    a.func(string("b"));
    a.func("b"); // ERROR: only one step is allowed
}
    需要注意，类型转换只允许一步。
    如果想抑制这种转换，需要在相应构造函数前添加关键字 explicit，且 explicit 构造函数只能直接初始化。

    explicit A(string s) { }
...
A a("a");
A a = "a"; // ERROR

    2. C++ 初始化

    C++ 的初始化分为 6 种：零初始化, 默认初始化, 值初始化, 直接初始化, 拷贝初始化, 列表初始化

    (1). 默认初始化

    • 块作用域内不使用任何初始值初始化一个非静态变量或数组
      • 未指明初值的静态 or 全局也会进行默认初始化，但由于零初始化相当于什么也没做
      • 对于非类变量，如一部分基本类型和指针，默认初始化的值是未定义的
      • 对于数组，其中每个元素都进行默认初始化
    • T object = new T，T object = new T()
    • 一个类类型显式使用默认初始化，或者一个类类型是另一个类的成员且没有在 constructor initialize list 中初始化。它们均会导致默认构造函数被调用。

    (2). 值初始化

    • 无名临时变量：T(), new T(), T{}, new T{}
    • 不使用初值定义局部静态变量
    • 数组初始化过程中提供的初值数量少于数组的大小时
    对于进行值初始化的类类型，如果其默认构造函数是用户自定义的，则进行默认初始化；如果是合成的，则进行零初始化。
