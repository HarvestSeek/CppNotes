    1.constructor

    1. 构造函数

    (1). 默认构造函数

     类通过默认构造函数执行默认初始化的过程，当一个类没有任何构造函数时，编译器会创建一个默认构造函数，它被称为合成的 (synthesized) 默认构造函数，其初始化行为：
     ① 如果存在类内值，则用类内值初始化成员；
     ② 否则，执行相应成员的默认初始化。
    当类中存在自定义的构造函数，需要通过 = default 显式声明合成的默认构造函数： 

    class A {   A() = default;  }
    合成的默认构造函数只适用于非常简单的类，应尽可能自定义默认构造函数。

    (2). 构造函数初始值列表 (initializer list)

    即通过如下形式初始化成员: 

    void func(<params>) : member1(expr), ... { }
    类中的所有成员根据它们在类的定义中出现的顺序执行初始化，每个成员再依次按照
      intializer list  ->  类内值  ->  执行默认初始化
    的顺序寻找初始化的机会。因此，构造函数初始值列表的必要性在于：
    • 在某些情况下，initializer list 是通过参数初始化 const 类型、引用类型、没有默认构造函数的类类型 的唯一机会；
    • 对比于先默认初始化再在构造函数内赋值的方式，一些类型会涉及初始化与赋值的底层效率问题。

    当一个成员的初始化需要用到另一个成员时，需要注意它们在类中定义的顺序。

    (3). 委托构造函数

    把自己的一部分职责委托给其他构造函数。 

    struct A {
    char a, b;
    A(char i, char j) : a(i), b(j) { }
    A(string s) : A(s[0], s[1]) { }
};

    (4). 隐式类类型转换

     只需要一个实参的构造函数被称为转换构造函数，它隐式地定义了由其他类型转换到此类型的规则，编译器会根据相应参数和构造函数自动创建一个临时量。 

    struct A {
    string label = "succ";
    A(string s) { }
    void func(A fake) { cout << fake.label; }
};

int main() {
    A a("a");
    a.func(string("b"));
    a.func("b"); // ERROR: only one step is allowed
}
    需要注意，类型转换只允许一步。
    如果想抑制这种转换，需要在相应构造函数前添加关键字 explicit，且 explicit 构造函数只能直接初始化。

    explicit A(string s) { }
...
A a("a");
A a = "a"; // ERROR

    2. C++ 初始化

    C++ 有多种初始化行为：零初始化, 默认初始化, 值初始化, 直接初始化, 拷贝初始化, 列表初始化

    (1). 默认初始化

    Syntax 

    T object;       (1)
new T           (2)

    • 具有静态存储期或自动存储期的变量，声明 without initializer
    • 通过 new 声明的具有动态存储期的变量，声明 without initializer
    • 基类或某个非静态数据成员，在某个 constructor 的 initializer list 没有被声明，而这个构造函数又被调用

    Effect

    • 如果 T 是 class type，调用默认构造函数为其赋初值
    • 如果 T 是 array type，其中的每个元素都执行默认初始化
    • 其他情况下，自动存储期内的对象被赋予一个 indeterminate value
    details

    (2). 值初始化

    Syntax 

    T(), T{}, T obj(), T obj{}                                             (1)
new T(), new T{},                                                      (2)
Class::Class(...) : member() {...}, Class::Class(...) : member{} {...} (3)

    • 命名对象或者无名的临时对象，通过 initializer with empty parenthesis or braces 初始化
    • 通过 new 声明的具有动态存储期的变量，通过 initializer with empty parenthesis or braces 初始化
    • 非静态数据成员或者基类，在 initializer list 中，通过 initializer with empty parenthesis or braces 初始化

    此外，在上述情形中

    • 如果 T 是聚合类型，且通过空的花括号初始化，则执行 aggregation-initialization
    • 如果 T 是类类型，且没有默认构造器，但有一个构造器使用了 std::initializer_list，则执行 list-initialization

    Effect

    • 如果 T 是 class type，
      • 若 T 没有默认构造函数 / 有用户提供的默认构造函数 / 有 deleted 构造函数，执行默认初始化
      • 若 T 的默认构造函数是隐式定义的 / 通过 = default 显式指明的 (编译期生成)，且如果这个构造函数是 trivial 的，执行零初始化；如果是 non-trivial 的，执行零初始化后还要执行默认初始化
    • 如果 T 是 array type，其中的每个元素都执行值初始化
    • 其他情况下，执行零初始化
    details
    trivial default constructor

    (3). 零初始化

    Syntax 

    static T object                                             (1)
T(), T t = {}, T{}                                          (2)
CharT array[n] = ""                                         (3)

    • 具有静态存储期的、被命名的变量
    • 在值初始化是提过的情形
    • 字符的数组通过字符串字面量初始化时，字符串长度不够，剩下的字符被零初始化

    Effect

    • 如果 T 是 scalar type，T 的初始值是整数常量 0 隐式转换为 T 后的值
    • 如果 T 是 non-union class type，所有的基类和非静态成员被零初始化，padding 用 0 比特填充，其所有构造函数都被忽略
    • 如果 T 是 array type，其中的每个元素都执行零初始化
    • 如果 T 是引用类型，什么也不做
    details
